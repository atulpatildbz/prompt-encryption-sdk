## DETOUR: L4 (network) vs L7 (HTTP) filters in Envoy
Status: COMPLETE (2026-06-05) — verified by user explaining the full arc back.
Goal: what each layer sees/does + when it fires; in A1, L4 writes the attested tag,
L7 reads it per request; WHY only L4 can write connection-scoped state (L7 write lands
in per-request scope and vanishes). Briefing emitted 2026-06-05 for a fresh conversation.

Now understands (verified by explaining back, taught against poc1/src/lib.rs + poc3 RESULTS):
- Scopes as nested lexical scope: L4 = connection (outer), L7 = stream (inner). Inner can
  READ outer (reads walk UP the ancestry: stream FS -> parent connection FS); inner cannot
  write into outer. set_property writes into the FS your CURRENT context owns, regardless of
  declared lifespan -> L4 write lands at connection scope, L7 write lands at (doomed) stream scope.
- Corrected mistake: user first thought L4 declares / L7 writes. Actual = L4 BOTH declares
  (declare_property @DownstreamConnection) AND writes (set_property); L7 ONLY reads (get_property).
- Why A1 breaks for the SDK: L4 writes at on_new_connection, BEFORE any HTTP request, so it
  cannot know the attestation verdict. Verdict is born at L7 (POST /_attest-connection), after
  on_new_connection, where the connection FS is unwritable -> A1 unusable -> pivot to A2.
- POC1 = A1 plumbing works IF verdict exists at conn-open (1 L4 write, N L7 reads, fresh conn
  ABSENT, no-writer listener ABSENT=fail-closed). POC3 = verdict-born-at-L7 forces A2.
- A2 fix + corrected imprecision: map is VM-LOCAL (thread-local per worker), NOT one global map.
  Connection pinned to one worker = one single-threaded VM -> same map across all its streams,
  separate map per worker -> no lock needed. Persists because the VM (worker-lived) outlives the
  stream; trade-off = no auto-GC, leaks 1 entry/closed-conn, needs L4 on_done eviction.
One-line arc: A1 stores verdict in a connection-lived container only L4 can write; SDK verdict is
born at L7 where that container is unreachable, so A2 uses a worker-lived container L7 can write,
trading Envoy's free GC for a manual-eviction chore.

Nuance added (2026-06-05): A1 is NOT broken/abandoned in principle — it's mismatched with WHERE
attestation runs. A1 viable iff verdict born at L4. So: attestation at L7 (SDK today) -> A2;
attestation at L4 (Solution B, pre-HTTP) -> A1 ideal (same filter attests AND writes tag, free GC).
The single change that makes A1 better than A2 again = move attestation to L4 (Solution B).
